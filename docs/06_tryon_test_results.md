# 試着テスト結果レポート: Nano Banana Pro

> テスト日: 2026-03-01

## テスト環境

- **モデル**: `gemini-3-pro-image-preview` (Nano Banana Pro)
- **中間処理**: なし（OpenPose, Graphonomy 等は一切不使用）
- **パイプライン**: `画像 + テキストプロンプト → Gemini API → 試着画像`
- **ライセンス**: 完全にクリーン（非商用コンポーネント不使用）

## テストモード

### Mode 1: clothing（商品画像から試着）
```
衣服商品画像 + 人物写真 → 試着画像
```
ECサイトの商品画像（平置き・ハンガー）を人物に着せるパターン。

### Mode 2: transfer（着用モデルから別人に着せ替え）
```
衣服を着たモデル + ターゲット人物 → ターゲットが同じ服を着た画像
```
PASTA-GAN++ が OpenPose + Graphonomy で行っていたパターンを、プロンプトのみで実現。

## テスト結果

### Test 1: clothing モード - 女性 + 黒Tシャツ
- **入力**: 160cm女性（茶色Tシャツ着用）+ 黒Tシャツ商品画像
- **結果**: 良好。体型・ポーズ保持、衣服のフィット自然
- **課題**: 顔が若干変化、アクセサリーが追加される

### Test 2: clothing モード - 男性 + 黒Tシャツ（初回プロンプト）
- **入力**: 178cm男性（がっしり体型）+ 黒Tシャツ商品画像
- **結果**: 体型保持は優秀
- **課題**: 構図が変化（全身→上半身寄り）、顔の特徴が変化

### Test 3: clothing モード - 男性 + 黒Tシャツ（顔保持改善プロンプト）
- **入力**: 同上
- **結果**: 顔の保持が改善
- **課題**: 元画像の測定テキスト（胸・腹・腰）が衣服デザインとして描画された

### Test 4: transfer モード - 女性の服を男性に着せ替え
- **入力**: 178cm男性 + 160cm女性（茶色Tシャツ+デニムショーツ着用）
- **結果**: 成功。衣服のデザイン・色・ロゴが正しく転写。男性体型にフィット
- **課題**: 測定テキストが残存、靴まで転写された

## Phase 2: クリーン画像テスト（測定テキスト・線なし）

> テスト日: 2026-03-01（Phase 1 に続き実施）

入力画像のノイズ（測定テキスト等）を除去したクリーン画像で再テスト。
すべて **transfer モード**（着用モデル→別人に着せ替え）で実施。

### Test 5: transfer モード - 男性→男性（標準体型・クリーン画像）
- **入力**: man1（ターゲット）+ man11（ソース: ボーダーセーター+カーキショーツ）
- **コマンド**: `python3 try_on_test.py --mode transfer --person test_data/person/man1.jpg --source test_data/person/man11.jpg`
- **結果**: ◎ 優秀。衣服のデザイン・色が正確に転写。体型・顔ともによく保持
- **課題**: 特になし

### Test 6: transfer モード - 子供→子供
- **入力**: kid2（ターゲット）+ kid1（ソース）
- **コマンド**: `python3 try_on_test.py --mode transfer --person test_data/person/kid2.jpg --source test_data/person/kid1.jpg`
- **結果**: ◎ 優秀。子供の体型に合わせた自然なフィット。衣服デザインの忠実度が高い
- **課題**: 特になし

### Test 7: transfer モード - 女性の服→動きのある男性（サッカー選手）
- **入力**: soccer1（ターゲット: アクションポーズ）+ woman1（ソース）
- **コマンド**: `python3 try_on_test.py --mode transfer --person test_data/person/soccer1.jpg --source test_data/person/woman1.jpg`
- **結果**: △ 衣服は転写されたが、顔の品質が低下。動きのあるポーズで精度が落ちる
- **課題**: 顔の再生成が目立つ。→ 中間処理（`--preprocess`）での改善を検証予定

### Test 8: transfer モード - スリム男性の服→大柄男性（体型差テスト）
- **入力**: man110（ターゲット: 大柄体型）+ man11（ソース: スリム体型、ボーダーセーター）
- **コマンド**: `python3 try_on_test.py --mode transfer --person test_data/person/man110.jpg --source test_data/person/man11.jpg`
- **結果**: ◎ 優秀。大柄体型にボーダーセーターが自然にストレッチして描画。体型差があっても適切にフィット
- **課題**: 特になし

## Phase 3: 高解像度 + アクションポーズテスト（Unsplash 画像）

> テスト日: 2026-03-01（Phase 2 に続き実施）

Unsplash から高解像度（1000px〜6000px幅）のアクションポーズ画像を取得してテスト。
**中間処理なし（Gemini 単体）** で実施。

### Test 9: transfer - 男性ダンス（激しい動き、高解像度）
- **入力**: man_dance1（ターゲット: ダンスポーズ、6193x9290px）+ man11（ソース）
- **結果**: ◎ ボーダーセーター+カーキショーツが正確に転写。激しいダンスポーズが完全維持
- **課題**: 特になし。**高解像度画像では Gemini 単体で十分**

### Test 10: transfer - 女性パンチポーズ + タイトトップ/ルーズボトム

- **結果**: ◎ ベージュクロップトップ+スウェットパンツが転写。タイト/ルーズの質感差も忠実
- **課題**: 特になし

### Test 11: transfer - 女性クラブダンス（暗所+カラフル照明）

- **入力**: woman_dance5（ターゲット: ダンスポーズ）+ man_suits1（ソース: ブラウンスーツ）

### IMAGE_SAFETY フィルター

- **露出の多い衣服がソース画像にある場合にブロックされる** Gemini の制約

### Phase 3 結論

1. **高解像度画像であれば Gemini 単体で十分な品質** - 中間処理（MediaPipe等）は不要
2. **Phase 2 の soccer1 の品質問題は低解像度（320px）が原因**だった
3. **手足の末端ポーズが変わることがある** - 顔を隠す手、脚の角度など
4. **背景がソース画像に引っ張られることがある** - 異性間着せ替え時に発生
5. **IMAGE_SAFETY フィルター** - 露出の多いソース画像はブロックされる

```bash
  --preprocess
```

## 発見・知見

1. **Nano Banana のみで仮想試着は実現可能** - OpenPose等の中間処理なしでも動作する
2. **transfer モードも動作** - PASTA-GAN++がやっていたモデル間着せ替えがプロンプトだけで可能
3. **プロンプトで品質が大きく変わる** - 顔保持の指示を強化すると改善する
4. **入力画像の品質が重要** - 測定テキスト等のノイズがそのまま結果に影響する
5. **ライセンス完全クリーン** - 非商用コンポーネント一切不使用
6. **クリーン画像で大幅改善** - 測定テキストなしの画像では顔・体型ともに保持が改善
7. **体型差への対応力が高い** - スリム→大柄など体型差があっても衣服が自然にフィット
8. ~~**動きのあるポーズは課題**~~ → **高解像度画像なら動きのあるポーズも問題なし**
9. **解像度が品質の最大要因** - 低解像度(320px)で品質低下、高解像度(1000px+)で ◎
10. **中間処理は基本不要** - Gemini 単体で十分な品質（高解像度前提）
11. **手足の末端ポーズが変わることがある** - 顔を隠す手、脚の角度など
12. **異性間着せ替えも可能** - 女性服→男性、男性スーツ→女性ともに対応
13. **IMAGE_SAFETY フィルター** - 露出の多いソース画像はブロックされる Gemini の制約
14. **衣服のフィット感を理解** - タイト/ルーズの質感差、重力による垂れなど物理的な表現が正確
