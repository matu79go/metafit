# META FIT プロジェクト履歴

## 1. プロジェクトの起源

META FITは「20年前の構想」に端を発するプロジェクト。元々はハードウェア・フォトブース型（日本のプリクラ風）として発想されたが、技術進化に伴いスマートフォンアプリへシフトした。

### 解決する課題
- オンラインファッション市場の返品率は30〜40%と深刻
- 消費者は試着できないため複数サイズを注文して試す
- 環境負荷も大きい

### ゴール
AIによるリアルな試着画像の即時生成で、購入前に「実際にどう見えるか」を確認可能にする。

## 2. 技術的アプローチの変遷

### Phase 1: PF-AFN (Pose Flow Attention Flow Network)
- **目的**: 2Dでの衣服ワーピング
- **手法**: ポーズ検出 → 衣服変形 → 合成
- **構成**:
  - AFWM (Attention Flow Warp Module): 衣服をポーズに合わせて変形
  - ResUnetGenerator: ブレンディングと精製
  - FlowNet2ベースのコスト計算
- **結果**: 基本的な2D試着は実現したが、品質に課題

### Phase 2: PIFu (Pixel-aligned Implicit Function)
- **目的**: 単一画像からの3D人体復元
- **手法**: 画像特徴抽出 → 3Dサンプル点投影 → 表面分類 → メッシュ生成
- **構成**:
  - Hourglass Network: 画像特徴抽出
  - MLP: 表面占有率予測
  - Marching Cubes: メッシュ生成
- **結果**: 3D復元は実現したが、試着への統合が複雑

### Phase 3: PASTA-GAN++
- **目的**: より高精度な仮想試着
- **手法**: 人体パーシング + キーポイント検出 + GAN生成
- **構成**:
  - OpenPose: 身体ポーズ検出 (18点)
  - Graphonomy: 人体セグメンテーション
  - PASTA-GAN++: 衣服生成ネットワーク
- **設定**:
  - 画像サイズ: 320×512
  - 対応部位: full / upper / lower
- **結果**: 標準体型では説得力のある結果。テクスチャ保持も改善。

### Phase 4: Gemini Nano Banana (現在)
- **目的**: 最新の生成AIによる完璧な仮想試着
- **手法**: 検証中
- **期待**: 従来のGANベースの限界を超える柔軟性と品質

## 3. Web フロントエンド実装

### 2D版
- TensorFlow.js + PoseNet でリアルタイムポーズ検出
- Canvas API で衣服オーバーレイ
- ブラウザ内で完結

### 3D版
- Three.js + WebGL で3Dレンダリング
- GLB形式の3Dモデル（人体・衣服）
- インタラクティブな操作（回転、ズーム）
- ARサポート

## 4. 成果と残存課題

### 成果
- 標準体型での説得力ある試着画像生成
- テクスチャとパターンの保持
- 複数の体型への基本対応（スリム体型は特に良好）
- 2D/3D両方のWebプロトタイプ

### 残存課題
| 課題 | 詳細 |
|------|------|
| 処理速度 | 1枚の生成に計算時間がかかる。リアルタイム体験には高速化必須 |
| 体型多様性 | 訓練データに少ない体型での結果劣化。全体型対応が商用化の条件 |
| 衣服忠実度 | 複雑なパターン・ロゴ・タイト衣服で細部が失われる傾向 |
| フィッティング | 体型に完全にフィットした結果を得ることが困難 |
